Windows-Terminal

Ich habe versucht, das Problem mit der Anwendung Windows Terminal aus dem Microsoft Store nachzustellen und festgestellt, dass die Verzögerungen aus dem WARP-Modus stammen, der den Text ohne GPU-Unterstützung rendert. Zusätzlich zu den Verzögerungen gibt es noch ein unangenehmes Merkmal: Die CPU wird ständig belastet, auch wenn das Windows-Terminal-Fenster versteckt oder zugeklappt ist. Ich habe beschlossen, Windows Terminal aufzugeben und einige alternative Anwendungen auszuprobieren. Hier ist meine Zusammenfassung:

  • Windows Terminal ⛔ Verzögerungen im Hintergrund
  • Wezterm ⛔ 500 MB VRAM
  • Tabby ⛔ 500 MB VRAM
  • Terminus ⛔ Erfordert eine Online-Registrierung bei der Installation
  • Alacritty ⛔ 200 MB VRAM, keine Tabs
  • ConEmu ⛔ 0 MB VRAM, gut 🐞 Aber es gibt Probleme mit dem Text-Rendering
  • VSCode ✅ 0 MB VRAM 🐞 Manchmal treten seltene, aber wilde Fehler beim Wiederherstellen der vorherigen Sitzung auf

Letztendlich habe ich mich für VSCode entschieden. Es scheint praktisch zu sein. Anwendungen, die viel VRAM verbrauchen, fallen sofort weg, da ich VRAM so gut wie möglich für KI-Aufgaben sparen möchte. Interessant ist, dass ich ausgerechnet VSCode gewählt habe - eine ziemlich schwere Anwendung, die auf Electron läuft. Andere Terminals bieten keinen Modus ohne GPU-Unterstützung. Wie man sieht, gibt es überall Bugs und Fehler. Die Frage ist, wo sie weniger sind und welche Fehler man einfach ignorieren kann. 🪲🤔💭
