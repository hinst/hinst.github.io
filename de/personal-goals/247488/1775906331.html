Terminal für Windows

Ich habe versucht, das Problem mit der Anwendung Windows Terminal aus dem Microsoft Store zu reproduzieren und festgestellt, dass die Verzögerungen vom WARP-Modus stammen, der den Text ohne GPU-Unterstützung rendert. Zusätzlich zu den Verzögerungen gibt es noch diese unangenehme Besonderheit: Die CPU-Auslastung ist konstant hoch, selbst wenn das Terminalfenster versteckt oder minimiert ist. Ich habe mich entschieden, Windows Terminal aufzugeben und einige andere Programme ausprobiert. Hier ist das Ergebnis:

  • Windows Terminal ⛔ Hintergrundverzögerungen
  • Wezterm ⛔ 500 MB VRAM
  • Tabby ⛔ 500 MB VRAM
  • Terminus ⛔ Erfordert Online-Registrierung bei der Anmeldung
  • Alacritty ⛔ 200 MB VRAM, keine Tabs
  • ConEmu ⛔ 0 MB VRAM, das ist gut 🐞 Aber es gibt Bugs bei der Textdarstellung
  • VSCode ✅ 0 MB VRAM 🐞 Es gibt seltene, aber wilde Bugs beim Wiederherstellen der vorherigen Sitzung

Letztendlich habe ich mich entschieden, VSCode🖥️ zu verwenden. Es scheint eigentlich praktisch zu sein. Programme, die viel VRAM verbrauchen, fallen sofort aus dem Rennen, da ich den VRAM maximal für KI spare. Interessant ist, dass ich mich gerade für VSCode entscheiden musste – eine ziemlich schwere Anwendung, die auf ⚛️ Electron basiert. Aber in anderen Terminals wurde kein ordentlicher Modus ohne GPU implementiert. Wie man so sagt: Bugs gibt es überall. Es existieren keine fehlerfreien Programme. Die Frage ist nur, wo sie weniger sind und welche Bugs man entspannt ignorieren kann. 🪲🤔💭
