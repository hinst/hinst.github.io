Orange Pi

I set up weekly backups. The files are stored on the same memory card as the entire system. It's not perfect, but it's an improvement from having no backups before. I also configured Uptime Kuma. However, the program has one drawback: it doesn't allow for a custom URL prefix; instead, you can only run it in the root domain. So, I won't be able to set up convenient access via my TailScale endpoint. Oh well, let's keep the Uptime monitor accessible only within my local network for security reasons.

I learned that a new version of Orange Pi has been released. But there are hardly any differences from my Orange Pi 2W, so there's no real need to buy it.

AI Models

I experimented with various AI models. For my computer, the LLM models of 24B Q6 and 32B Q4 work best. Larger models start to lag, while smaller ones are suitable for maximum speed when needed. There are numerous AI models available online, so it would take a lot of time to test them all. I can keep experimenting for a long time.
