-= Personal Website =- 🌐

Last week I migrated my website to PostgreSQL, and this week I found an interesting bug 🐞 It turns out that the Go driver for PostgreSQL doesn't have a connection timeout. When my program tries to open a new connection to the database and the connection limit is already reached, it goes into an infinite wait. Why - only the author of the library knows. You can't just set a timeout; you have to write clever code with contexts, as shown in the screenshot.
