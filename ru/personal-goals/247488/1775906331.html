Терминал для Windows

Пытался воспроизвести проблему с прогой Windows Terminal из Microsoft Store, и обнаружил, что тормоза исходят из режима WARP, который отрисовывает текст без помощи GPU. В добавок к тормозам, есть ещё такая неприятная особенность: нагрузка на процессор идёт постоянно, даже если окно терминала скрыто или свёрнуто. Решил отказаться от Windows Terminal, и попробовал ещё несколько разных прог. Вот итог:

  • Windows Terminal ⛔ Тормоза в фоне
  • Wezterm ⛔ 500 MB VRAM
  • Tabby ⛔500 MB VRAM
  • Terminus ⛔ Требует регистрацию онлайн аккаунта при установке
  • Alacritty ⛔ 200 MB VRAM, нету табов
  • ConEmu ⛔ 0 MB VRAM, это хорошо 🐞 Но есть баги с отображением текста
  • VSCode ✅ 0 MB VRAM 🐞 Есть редкие, но дикие баги с восстановлением предыдущей сессии

В итоге, я решил использовать VSCode🖥️. Вроде бы, удобно. Проги, которые используют много VRAM - сразу отпадают, ибо я максимально экономлю VRAM для AI. Занятно, что пришлось выбрать именно VSCode - довольно таки тяжёлая прога, которая едет на ⚛️ Электроне. Но в другие терминалы нормальный режим без GPU не завезли. Как говорится, баги есть везде. Программ без багов не существует. Вопрос в том, где их меньше, и какие баги можно спокойно пережить. 🪲🤔💭
